Практичні завдання (7 клас): інформаційна модель у табличному процесорі

Створи інформаційну модель (таблиці + формули + аналіз + діаграми) у Excel або Google Sheets. Обери тему та виконай кроки як у прикладі.

Excel / Google Sheets Формули: SUM, MAX, AVERAGE Діаграми 3 аркуші
Як виконувати (коротко)
  • 1. Створи файл і 3 аркуші: Довідники, Дані, Аналіз
  • 2. Внеси дані з блоку “Набір даних”
  • 3. Додай формули та підсумки
  • 4. Побудуй діаграми
  • 5. Зроби висновки (2–4 речення)
Порада

Для цін зручно робити абсолютні посилання (наприклад $B$2), а для кількостей — відносні. Так можна швидко копіювати формули.

Назви діаграми та підпиши осі. Це частина “зрозумілої інформаційної моделі”.

Обери тему
  • Мінікафе
  • Інтернет-магазин
  • Блогер
  • Стартап
  • Кіберспорт
restaurant Практичне завдання 1: Мінікафе «Smart Snack»

У школі працює учнівське мінікафе. Побудуй модель, яка показує: дохід, витрати, прибуток, найкращий товар і найкращий день.

Що треба зробити (в Sheets/Excel)
  1. Аркуш “Довідники”: створи таблицю товарів + стовпець Прибуток_з_1шт = Ціна_продажу − Закупівля.
  2. Аркуш “Дані”: внеси таблицю продажів. Додай стовпці: Дохід_дня, Витрати_дня, Прибуток_дня.
  3. Формули:
    • Дохід_дня = сума(кількість × ціну продажу для кожного товару)
    • Витрати_дня = сума(кількість × закупівельну ціну)
    • Прибуток_дня = Дохід_дня − Витрати_дня
  4. Аркуш “Аналіз”:
    • Загальний дохід і прибуток за тиждень (SUM).
    • Найприбутковіший день (MAX).
    • Найпопулярніший товар за тиждень (сума кількості по товару + MAX).
    • Середній прибуток за день (AVERAGE).
  5. Діаграми:
    • Стовпчаста: прибуток по днях.
    • Кругова: частка продажів товарів за тиждень.
    • Лінійна: дохід по днях.
  6. Висновок: 2–4 речення (що продавати більше, у які дні готуватися).

Додатково (за бажанням): зроби колонку Знижка 10% та порівняй прибуток “до/після”.

Що треба зробити (в Sheets/Excel)
  1. Аркуш “Довідники”: таблиця товарів + стовпці: Комісія_з_1шт = Ціна × (Комісія%/100), Чистий_прибуток_з_1шт = Ціна − Собівартість − Комісія_з_1шт.
  2. Аркуш “Дані”: внеси замовлення. Додай стовпці: Виручка_дня, Комісія_дня, Чистий_прибуток_дня.
  3. Формули:
    • Виручка_дня = сума(кількість × ціну)
    • Комісія_дня = сума(кількість × комісія_з_1шт)
    • Чистий_прибуток_дня = сума(кількість × чистий_прибуток_з_1шт)
  4. Аркуш “Аналіз”:
    • Загальна виручка та чистий прибуток за 5 днів (SUM).
    • Найкращий день за прибутком (MAX).
    • Топ-товар за виручкою та топ-товар за прибутком (суми по товарах).
    • Середній прибуток за день (AVERAGE).
  5. Діаграми:
    • Стовпчаста: прибуток по днях.
    • Кругова: частка прибутку по товарах.
    • Лінійна: виручка по днях.
  6. Висновок: 2–4 речення (які товари вигідні, які “продаються багато, але заробляють мало”).

Додатково: запропонуй 1 зміну (ціна/комісія/акція) і спрогнозуй, як зміниться прибуток.

Що треба зробити (в Sheets/Excel)
  1. Аркуш “Довідники”: внеси тарифи та назви їх зрозуміло.
  2. Аркуш “Дані”: внеси статистику та додай стовпці: Доход_з_переглядів, Доход_з_реклами, Загальний_дохід.
  3. Формули:
    • Доход_з_переглядів = Перегляди_тис × Тариф_за_1000
    • Доход_з_реклами = Реклама × Оплата_за_1
    • Загальний_дохід = Перегляди + Реклама + Донати
  4. Аркуш “Аналіз”:
    • Загальний дохід за 4 тижні (SUM).
    • Найкращий тиждень (MAX).
    • Яке джерело доходу найбільше (сума по кожному джерелу).
    • Середній дохід за тиждень (AVERAGE).
  5. Діаграми:
    • Лінійна: загальний дохід по тижнях.
    • Стовпчаста: доходи по джерелах (сума за 4 тижні).
    • Кругова: частка джерел доходу.
  6. Висновок: 2–4 речення (на чому блогеру вигідніше зосередитись).

Додатково: спрогнозуй 5-й тиждень, якщо перегляди зростуть на 10%, а реклами буде на 1 більше.

Що треба зробити (в Sheets/Excel)
  1. Аркуш “Довідники”: внеси ціни підписок (Basic, Pro).
  2. Аркуш “Дані”: внеси план на 4 місяці та додай стовпці: Дохід, Витрати, Прибуток, Накопичений_результат.
  3. Формули:
    • Дохід = Basic×79 + Pro×149
    • Витрати = Реклама + Сервер + Дизайн
    • Прибуток = Дохід − Витрати
    • Накопичений_результат = попередній накопичений + поточний прибуток
  4. Аркуш “Аналіз”:
    • Загальний прибуток за 4 місяці (SUM).
    • Найкращий місяць (MAX).
    • У який місяць накопичений результат стане > 0.
    • Середній прибуток (AVERAGE).
  5. Діаграми:
    • Стовпчаста: прибуток по місяцях.
    • Лінійна: накопичений результат.
    • Стовпці: дохід vs витрати.
  6. Висновок: 2–4 речення (що треба покращити, щоб швидше вийти “в плюс”).

Додатково: зроби сценарій “Pro +20%” у 4-му місяці та порівняй прибуток.

Що треба зробити (в Sheets/Excel)
  1. Аркуш “Довідники”: внеси правила (очки, призові, витрати).
  2. Аркуш “Дані”: внеси матчі та додай стовпці: Очки, Призові, Витрати, Прибуток, KDA.
  3. Формули:
    • Очки: якщо результат Win → 3, інакше 0.
    • Призові: якщо Win → 700, інакше 0.
    • Витрати = 250 для кожного матчу.
    • Прибуток = Призові − Витрати.
    • KDA = (Кілли + Асисти) / Дізи.
  4. Аркуш “Аналіз”:
    • Загальні очки (SUM) та Win-rate (перемоги/всі матчі).
    • Середній KDA (AVERAGE).
    • Найкращий матч за KDA (MAX).
    • Загальний прибуток за сезон (SUM).
  5. Діаграми:
    • Стовпчаста: очки по матчах.
    • Лінійна: KDA по матчах.
    • Стовпці: призові vs витрати.
  6. Висновок: 2–4 речення (що покращити команді: стабільність/виживання/командна гра).

Додатково: запропонуй правило бонусу (наприклад +100 грн за KDA > 3) і перерахуйте прибуток.

Критерії оцінювання (НУШ)

Оцінювання компетентнісне: важливо не лише “порахувати”, а й пояснити висновки та показати охайну, логічну модель. Рівні можна використати як 12-бальну шкалу (1–3 / 4–6 / 7–9 / 10–12).

account_tree1) Структура моделі
Аркуші, назви, логіка “довідники → дані → аналіз”.
ПочатковийСереднійДостатнійВисокий
  • Початковий: частково створені таблиці, плутанина з аркушами.
  • Середній: 3 аркуші є, але структура або назви не дуже зрозумілі.
  • Достатній: чітко розділено довідники/дані/аналіз, охайні заголовки.
  • Високий: додано форматування, фільтри, логічні блоки, зручність читання.
functions2) Формули й обчислення
Правильність формул, копіювання, посилання на довідники.
ПочатковийСереднійДостатнійВисокий
  • Початковий: є помилки у формулах або багато ручних підрахунків.
  • Середній: базові формули працюють, але не всюди/є неточності.

Відповіді на питання: найкращий день/товар, підсумки, середні значення.
ПочатковийСереднійДостатнійВисокий

  • Середній: є підсумки, але не всі ключові висновки.
  • Достатній: знайдено топ-показники, підсумки за період, середнє.
  • Високий: є порівняння/пояснення “чому так”, додатковий сценарій “що як”.
insert_chart4) Візуалізація й висновок
Діаграми + короткий текстовий висновок (2–4 речення).
ПочатковийСереднійДостатнійВисокий
  • Початковий: діаграм немає або вони без підписів.
  • Середній: 1–2 діаграми, але не дуже читабельні.
  • Достатній: 3 діаграми з назвами й осями + висновок.
  • Високий: діаграми доречні, охайні, висновок аргументований (спирається на числа).
Продукт діяльності учня

Файл таблиці з 3 аркушами + 3 діаграми + короткий висновок. За бажанням — “що як” сценарій (знижка/зростання/комісія тощо).
